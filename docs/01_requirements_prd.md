# Gitread — Product Requirements Document
Version: v1.0, 2026-04-03

---

## 1. Problem Statement

Developers spend years building public GitHub histories, but those histories are unreadable to the people who matter most — recruiters and hiring managers. Raw GitHub profiles reward activity quantity, not quality, and bury the signal in noise: fork counts, star graphs, and language percentages that mean nothing without context. Developers have no easy way to turn their commit history into a coherent professional narrative, and recruiters have no fast way to evaluate a developer's actual depth without reading code.

Gitread solves this for both sides: it reads a developer's public GitHub activity and translates it into a structured, honest engineering profile — generated by AI, grounded in real data, shareable in seconds.

---

## 2. Personas

### Primary — The Developer (Showcase Mode)
A software engineer, student, or open source contributor who wants to present their GitHub activity professionally. They are comfortable sharing their GitHub username but do not want to manually write a bio or curate their profile. They want a shareable link they can drop in a CV, LinkedIn, or portfolio.

### Secondary — The Recruiter (Evaluation Mode)
A technical recruiter or hiring manager evaluating candidates. They have 30–60 seconds per candidate. They need a fast, scannable summary of what a developer has actually built, what stack they use, and whether they are a fit for a specific role. They may paste a job description to score the match.

### Tertiary — The Developer (Self-Tracking Mode)
A developer using Gitread periodically to track their own growth — which languages they are investing in, whether their commit consistency is improving, and what to build next to level up their profile.

---

## 3. Solution Overview

Gitread is a public web application. A user enters any GitHub username. Gitread fetches that user's public repositories, commit activity, language distribution, and star history from the GitHub REST API, then passes the structured data to an LLM to generate a profile. The profile includes a recruiter-ready summary, a developer archetype label, a language breakdown chart, a 90-day activity heatmap, a top repos showcase, and a suggested next project. Every profile is accessible at `gitread.dev/u/{username}` — no login required to view or share.

The unique differentiator is the **Job Description Match** feature: a recruiter or developer pastes a job URL or description, and Gitread scores the developer's profile against it (0–100), identifies matched and missing skills, and recommends what to build to close the gap.

---

## 4. MVP Scope

### Profile Generation
- Accept any public GitHub username as input
- Fetch public repos, commit counts, languages, stars, forks, and account age
- Generate an AI profile card: title, bio, 3–5 key strengths, tailored to actual stack and activity
- Assign a developer archetype (e.g., "The Fullstack Shipper", "The CLI Craftsman")
- Generate an AI one-line description for each of the top 5 repos

### Data Visualisation
- Language breakdown donut chart weighted by stars (not repo count)
- 90-day commit frequency activity heatmap
- Stats at a glance: total stars, forks, public repos, account age, commit count

### Discovery & Sharing
- Shareable public link: `gitread.dev/u/{username}` — no login required
- Profile accessible without authentication

### Growth Intelligence
- Skill progression: language and framework usage trend across the last 6 months
- Suggested next project: AI recommendation based on visible stack gaps

### Job Description Match (Differentiator)
- Input: job URL or pasted job description text
- Output: match score (0–100), matched skills list, missing skills list, recommended project to close the gap

---

## 5. Out of Scope (for MVP)

- User accounts, authentication, or saved sessions
- Private repository analysis (OAuth GitHub integration)
- Payment processing or subscription tiers
- Email notifications or digest emails
- Mobile native app (web only)
- Team or organisation-level profiles
- Multilingual UI (English only)
- Advanced recruiter dashboard or bulk candidate comparison
- Real-time webhook updates from GitHub
- Embedding Gitread profiles in external sites (iframe/widget)
- Exporting profiles to PDF or other formats
- API access for third-party developers

---

## 6. Success Criteria

- Any valid public GitHub username generates a complete profile in under 15 seconds
- Profile page loads and is readable on mobile without horizontal scrolling
- Job match score is returned within 10 seconds of pasting a job description
- Shareable link works for any generated profile with no login prompt
- AI-generated content is factually consistent with visible GitHub data (zero fabricated repos or languages)
- Activity heatmap reflects actual commit counts for the 90-day window
- Language breakdown reflects weighted-by-stars distribution, not raw repo count
